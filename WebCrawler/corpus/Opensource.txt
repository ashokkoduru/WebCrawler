<div class="mw-content-ltr" dir="ltr" id="mw-content-text" lang="en"><div class="hatnote" role="note">This article is about the general production and development model. For its application to software, see <a class="mw-redirect" href="/wiki/Open-source_software_development_model" title="Open-source software development model">Open-source software development model</a>. For other uses, see <a class="mw-disambig" href="/wiki/Open_source_(disambiguation)" title="Open source (disambiguation)">Open source (disambiguation)</a>.</div>


<p>The <b>open-source model</b> is a more decentralized model of production, in contrast with more centralized models of <a href="/wiki/Software_developer" title="Software developer">development</a> such as those typically used in commercial software companies. Scientists view the open-source model as a case of <a href="/wiki/Open_collaboration" title="Open collaboration">open collaboration</a>. A main principle of <a href="/wiki/Open-source_software_development" title="Open-source software development">open-source software development</a> is <a href="/wiki/Peer_production" title="Peer production">peer production</a>, with products such as source code, "<a class="mw-redirect" href="/wiki/Blueprints" title="Blueprints">blueprints</a>", and documentation available to the public at no cost. The <a class="mw-redirect" href="/wiki/Open-source_movement" title="Open-source movement">open-source movement</a> in software began as a response to the limitations of proprietary code. Since, its principles spread across different fields to become what is known as <a href="/wiki/Open_collaboration" title="Open collaboration">open collaboration</a>. This model is also used for the development of <a class="mw-redirect" href="/wiki/Open-source_appropriate_technologies" title="Open-source appropriate technologies">open-source appropriate technologies</a>, and open-source drug discovery.</p>
<p></p>

<h2><span class="mw-headline" id="History">History</span></h2>
<div class="hatnote" role="note">Main article: <a href="/wiki/History_of_free_and_open-source_software" title="History of free and open-source software">History of free and open-source software</a></div>

<p>In production and development, <b>open source</b> as a <a href="/wiki/Software_development" title="Software development">development</a> model promotes universal access via an <a href="/wiki/Open-source_license" title="Open-source license">open-source</a> or <a href="/wiki/Free_license" title="Free license">free license</a> to a product's design or blueprint, and universal redistribution of that design or blueprint, including subsequent improvements to it by anyone. Before the phrase <i>open source</i> became widely adopted, developers and producers used a variety of other terms. <i>Open source</i> gained hold with the rise of the <a href="/wiki/Internet" title="Internet">Internet</a>, and the attendant need for massive retooling of the computing <a href="/wiki/Source_code" title="Source code">source code</a>. Opening the source code enabled a self-enhancing diversity of production models, communication paths, and interactive communities. The <a href="/wiki/Open-source_software_movement" title="Open-source software movement">open-source software movement</a> arose to clarify the environment that the new <a href="/wiki/Copyright" title="Copyright">copyright</a>, <a class="mw-redirect" href="/wiki/Licensing" title="Licensing">licensing</a>, <a href="/wiki/Domain_name" title="Domain name">domain</a>, and consumer issues created.</p>
<p>Generally, open source refers to a <a href="/wiki/Computer_program" title="Computer program">computer program</a> in which the <a href="/wiki/Source_code" title="Source code">source code</a> is available to the general public for use and/or modification from its original design. Open-source code is meant to be a collaborative effort, where programmers improve upon the source code and share the changes within the community. Typically this is not the case, and code is merely released to the public under some license. Others can then download, modify, and publish their version (fork) back to the community. Today you find more projects with forked versions than unified projects worked by large teams.</p>
<p>Many large formal institutions have sprung up to support the development of the open-source movement, including the <a href="/wiki/Apache_Software_Foundation" title="Apache Software Foundation">Apache Software Foundation</a>, which supports projects such as the open-source framework behind <a href="/wiki/Big_data" title="Big data">big data</a> <a href="/wiki/Apache_Hadoop" title="Apache Hadoop">Apache Hadoop</a> and an open-source <a class="mw-redirect" href="/wiki/HTTP" title="HTTP">HTTP</a> server <a href="/wiki/Apache_HTTP_Server" title="Apache HTTP Server">Apache HTTP</a>.</p>
<h3><span class="mw-headline" id="Sharing_of_technological_information_before_the_internet">Sharing of technological information before the internet</span></h3>
<p>The sharing of technological information predates the <a href="/wiki/Internet" title="Internet">Internet</a> and the personal computer considerably. For instance, in the early years of automobile development a group of capital <a href="/wiki/Monopoly" title="Monopoly">monopolists</a> owned the rights to a <a class="mw-redirect" href="/wiki/2_cycle" title="2 cycle">2-cycle</a> gasoline engine patent originally filed by <a href="/wiki/George_B._Selden" title="George B. Selden">George B. Selden</a>. By controlling this patent, they were able to monopolize the industry and force car manufacturers to adhere to their demands, or risk a lawsuit. In 1911, independent automaker <a href="/wiki/Henry_Ford" title="Henry Ford">Henry Ford</a> won a challenge to the Selden patent. The result was that the Selden patent became virtually worthless and a new association (which would eventually become the <a class="mw-redirect" href="/wiki/Motor_Vehicle_Manufacturers_Association" title="Motor Vehicle Manufacturers Association">Motor Vehicle Manufacturers Association</a>) was formed. The new association instituted a cross-licensing agreement among all US auto manufacturers: although each company would develop technology and file patents, these patents were shared openly and without the exchange of money among all the manufacturers. By the time the US entered <a href="/wiki/World_War_II" title="World War II">World War II</a>, 92 Ford patents and 515 patents from other companies were being shared among these manufacturers, without any exchange of money (or lawsuits).</p>
<h3><span class="mw-headline" id="Sharing_of_information_with_computers">Sharing of information with computers</span></h3>
<p>In an open call process analogous to later Internet-era <a class="mw-redirect" href="/wiki/Open_standards" title="Open standards">open standards</a> processes, <a href="/wiki/ARPANET" title="ARPANET">ARPANET</a> researchers used an open "<a href="/wiki/Request_for_Comments" title="Request for Comments">Request for Comments</a>" (RFC) process to encourage feedback in early telecommunication network protocols. This collaborative process of the 1960s led to the birth of the early Internet in 1969.</p>
<p>Early instances of the free sharing of source code include <a href="/wiki/IBM" title="IBM">IBM</a>'s source releases of its <a href="/wiki/Operating_system" title="Operating system">operating systems</a> and other programs in the 1950s and 1960s, and the <a href="/wiki/SHARE_(computing)" title="SHARE (computing)">SHARE</a> user group that formed to facilitate the exchange of software.</p>
<p>In a foreshadowing of the Internet, software with source code included became available on <a href="/wiki/Bulletin_board_system" title="Bulletin board system">BBS</a> networks in the 1980s. This was sometimes a necessity; distributing software written in <a href="/wiki/BASIC" title="BASIC">BASIC</a> and other <a href="/wiki/Interpreted_language" title="Interpreted language">interpreted languages</a> can only be distributed as source code as there is no separate portable executable binary to distribute.</p>
<p>Example of BBS systems and networks that gathered source code, and set up boards specifically to discuss its modification includes <a href="/wiki/WWIV" title="WWIV">WWIV</a>, developed initially in BASIC by <a class="mw-redirect" href="/wiki/Wayne_Bell_(computer_specialist)" title="Wayne Bell (computer specialist)">Wayne Bell</a>. A culture of modifying his software and distributing the modifications, grew up so extensively that when the software was ported to first <a href="/wiki/Pascal_(programming_language)" title="Pascal (programming language)">Pascal</a>, then <a href="/wiki/C%2B%2B" title="C++">C++</a>, its source code continued to be distributed to registered users, who would share mods and compile their own versions of the software. This may have contributed to its being a dominant system and network, despite being outside the <a class="mw-redirect" href="/wiki/Fidonet" title="Fidonet">Fidonet</a> umbrella that was shared by so many other BBS makers.</p>
<p>The sharing of source code on the Internet began when the Internet was relatively primitive, with software distributed via <a href="/wiki/UUCP" title="UUCP">UUCP</a>, <a href="/wiki/Usenet" title="Usenet">Usenet</a>, <a href="/wiki/Internet_Relay_Chat" title="Internet Relay Chat">IRC</a>, and <a href="/wiki/Gopher_(protocol)" title="Gopher (protocol)">Gopher</a>. <a class="mw-redirect" href="/wiki/BSD" title="BSD">BSD</a>, for example, was first widely distributed by posts to comp.os.linux on the Usenet, which is also where its development was discussed. <a href="/wiki/Linux" title="Linux">Linux</a> followed in this model.</p>
<h3><span class="mw-headline" id="The_emergence_of_the_.22open_source.22_term">The emergence of the "open source" term</span></h3>
<p>The term "open source" was first proposed by a group of people in the <a href="/wiki/Free_software_movement" title="Free software movement">free software movement</a> who were critical of the political agenda and moral philosophy implied in the term "free software" and sought to reframe the discourse to reflect a more commercially minded position. In addition, the ambiguity of the term "free software" was seen as discouraging business adoption. The group included <a href="/wiki/Christine_Peterson" title="Christine Peterson">Christine Peterson</a>, Todd Anderson, <a href="/wiki/Larry_Augustin" title="Larry Augustin">Larry Augustin</a>, <a href="/wiki/Jon_Hall_(programmer)" title="Jon Hall (programmer)">Jon Hall</a>, Sam Ockman, <a href="/wiki/Michael_Tiemann" title="Michael Tiemann">Michael Tiemann</a> and <a href="/wiki/Eric_S._Raymond" title="Eric S. Raymond">Eric S. Raymond</a>. Peterson suggested "open source" at a meeting held at <a href="/wiki/Palo_Alto,_California" title="Palo Alto, California">Palo Alto, California</a>, in reaction to <a class="mw-redirect" href="/wiki/Netscape_Communications_Corporation" title="Netscape Communications Corporation">Netscape</a>'s announcement in January 1998 of a source code release for <a href="/wiki/Netscape_Navigator" title="Netscape Navigator">Navigator</a>. <a href="/wiki/Linus_Torvalds" title="Linus Torvalds">Linus Torvalds</a> gave his support the following day, and Phil Hughes backed the term in <i><a href="/wiki/Linux_Journal" title="Linux Journal">Linux Journal</a></i>. <a href="/wiki/Richard_Stallman" title="Richard Stallman">Richard Stallman</a>, the founder of the free software movement, initially seemed to adopt the term, but later changed his mind. Netscape released its source code under the <a href="/wiki/Netscape_Public_License" title="Netscape Public License">Netscape Public License</a> and later under the <a href="/wiki/Mozilla_Public_License" title="Mozilla Public License">Mozilla Public License</a>.</p>
<p>Raymond was especially active in the effort to popularize the new term. He made the first public call to the free software community to adopt it in February 1998. Shortly after, he founded The <a href="/wiki/Open_Source_Initiative" title="Open Source Initiative">Open Source Initiative</a> in collaboration with <a href="/wiki/Bruce_Perens" title="Bruce Perens">Bruce Perens</a>.</p>
<p>The term gained further visibility through an event organized in April 1998 by technology publisher <a href="/wiki/Tim_O%27Reilly" title="Tim O'Reilly">Tim O'Reilly</a>. Originally titled the "Freeware Summit" and later known as the "Open Source Summit", the event was attended by the leaders of many of the most important free and open-source projects, including Linus Torvalds, <a href="/wiki/Larry_Wall" title="Larry Wall">Larry Wall</a>, <a href="/wiki/Brian_Behlendorf" title="Brian Behlendorf">Brian Behlendorf</a>, <a href="/wiki/Eric_Allman" title="Eric Allman">Eric Allman</a>, <a href="/wiki/Guido_van_Rossum" title="Guido van Rossum">Guido van Rossum</a>, <a href="/wiki/Michael_Tiemann" title="Michael Tiemann">Michael Tiemann</a>, <a href="/wiki/Paul_Vixie" title="Paul Vixie">Paul Vixie</a>, <a href="/wiki/Jamie_Zawinski" title="Jamie Zawinski">Jamie Zawinski</a>, and Eric Raymond. At that meeting, alternatives to the term "free software" were discussed. Tiemann argued for "sourceware" as a new term, while Raymond argued for "open source". The assembled developers took a vote, and the winner was announced at a press conference the same evening.</p>
<p>"Open source" has never managed to supersede entirely the older term "free software", giving rise to the combined term <a href="/wiki/Free_and_open-source_software" title="Free and open-source software">free and open-source software</a> (FOSS).</p>
<h2><span class="mw-headline" id="Economics">Economics</span></h2>
<div class="hatnote" role="note">Main article: <a href="/wiki/Open-source_economics" title="Open-source economics">Open-source economics</a></div>

<p>Most economists agree that open-source candidates have an <a href="/wiki/Information_good" title="Information good">information good</a> (also termed "knowledge good") aspect. In general, this suggests that the original work involves a great deal of time, money, and effort. However, the cost of reproducing the work is very low, so that additional users may be added at zero or near zero cost – this is referred to as the <a href="/wiki/Marginal_cost" title="Marginal cost">marginal cost</a> of a product. <a href="/wiki/Copyright" title="Copyright">Copyright</a> creates a monopoly so the price charged to consumers can be significantly higher than the marginal cost of production. This allows the author to recoup the cost of making the original work, without needing to find a single customer that can bear the entire cost. Conventional copyright thus creates access costs for consumers who value the work more than the marginal cost but less than the initial production cost. Access costs also pose problems for authors who wish to create a <a href="/wiki/Derivative_work" title="Derivative work">derivative work</a>—such as a copy of a software program modified to fix a bug or add a feature, or a <a href="/wiki/Remix" title="Remix">remix</a> of a song—but are unable or unwilling to pay the copyright holder for the right to do so.</p>
<p>Being organized effectively as a <a class="mw-redirect" href="/wiki/Consumers%27_cooperative" title="Consumers' cooperative">consumers' cooperative</a>, the idea of open source is to eliminate the access costs of the consumer and the creators of derivative works by reducing the restrictions of copyright. Basic economic theory predicts that lower costs would lead to higher consumption and also more frequent creation of derivative works. Additionally some proponents argue that open source also relieves society of the administration and enforcement costs of copyright. Organizations such as <a href="/wiki/Creative_Commons" title="Creative Commons">Creative Commons</a> have websites where individuals can file for alternative "licenses", or levels of restriction, for their works. These self-made protections free the general society of the costs of policing copyright infringement. Thus, on several fronts, there is an efficiency argument to be made on behalf of open-sourced goods.</p>
<p>However, others argue that because consumers do not pay for the copies, creators are unable to recoup the initial cost of production, and thus have no economic incentive to create in the first place. By this argument, consumers would lose out because some of the goods they would otherwise purchase would not be available at all. In practice, content producers can choose whether to adopt a proprietary license and charge for copies, or an open license. Some goods which require large amounts of professional research and development, such as the <a href="/wiki/Pharmaceutical_industry" title="Pharmaceutical industry">pharmaceutical industry</a> (which depends largely on <a href="/wiki/Patent" title="Patent">patents</a>, not copyright for intellectual property protection) are almost exclusively proprietary, although increasingly sophisticated technologies are being developed on open-source principles.</p>
<p>There is growing evidence, however, that open-source development creates enormous value. For example, in the context of <a href="/wiki/Open-source_hardware" title="Open-source hardware">open-source hardware</a> design, digital designs are shared for free and then anyone with access to digital manufacturing technologies (e.g. <a class="mw-redirect" href="/wiki/RepRap" title="RepRap">RepRap</a> 3D printers) can replicate the product for the cost of materials. The original sharer gains feedback and potentially improvements on the original design from the <a href="/wiki/Peer_production" title="Peer production">peer production</a> community.</p>
<h3><span class="mw-headline" id="Alternatives">Alternatives</span></h3>
<div class="hatnote" role="note">Further information: <a href="/wiki/Business_models_for_open-source_software" title="Business models for open-source software">Business models for open-source software</a></div>

<p>Alternative arrangements have also been shown to result in good creation outside of the proprietary model. Examples include:</p>
<ul>
<li>Creation for its own sake – For example, <a href="/wiki/Wikipedia" title="Wikipedia">Wikipedia</a> editors add content for recreation. Artists have a drive to create. Both communities benefit from free starting material.</li>
<li>Voluntary after-the-fact donations – used by <a href="/wiki/Shareware" title="Shareware">shareware</a>, <a class="mw-redirect" href="/wiki/Street_performer" title="Street performer">street performers</a>, and <a href="/wiki/Public_broadcasting" title="Public broadcasting">public broadcasting</a> in the United States.</li>
<li>Patron – For example, <a href="/wiki/Open_access" title="Open access">open access</a> publishing relies on institutional and government funding of research faculty, who also have a professional incentive to publish for reputation and career advancement. Works of the U.S. federal government are automatically released into the <a href="/wiki/Public_domain" title="Public domain">public domain</a>.</li>
<li><a href="/wiki/Freemium" title="Freemium">Freemium</a> – Give away a limited version for free and charge for a premium version (perhaps using a <a class="mw-redirect" href="/wiki/Dual_license" title="Dual license">dual license</a>)</li>
<li>Give away the product and charge something related – Charge for support of open-source <a href="/wiki/Enterprise_software" title="Enterprise software">enterprise software</a>, give away music but charge for concert admission</li>
<li>Give away work in order to gain market share – Used by artists, in corporate software to spoil a dominant competitor (for example in the <a href="/wiki/Browser_wars" title="Browser wars">browser wars</a> and the <a class="mw-redirect" href="/wiki/Android_operating_system" title="Android operating system">Android operating system</a>)</li>
<li>For own use – Businesses or individual software developers often create software to solve a problem, bearing the full cost of initial creation. They will then open source the solution, and benefit from the improvements others make for their own needs. Communalizing the maintenance burden distributes the cost across more users; <a href="/wiki/Free_rider_problem" title="Free rider problem">free riders</a> can also benefit without undermining the creation process.</li>
</ul>
<h2><span class="mw-headline" id="Applications">Applications</span></h2>

<p>Social and political views have been affected by the growth of the concept of open source. Advocates in one field often support the expansion of open source in other fields. But <a class="mw-redirect" href="/wiki/Eric_Raymond" title="Eric Raymond">Eric Raymond</a> and other founders of the <a class="mw-redirect" href="/wiki/Open-source_movement" title="Open-source movement">open-source movement</a> have sometimes publicly argued against speculation about applications outside software, saying that strong arguments for software openness should not be weakened by overreaching into areas where the story may be less compelling. The broader impact of the open-source movement, and the extent of its role in the development of new information sharing procedures, remain to be seen.</p>
<p>The <a class="mw-redirect" href="/wiki/Open-source_movement" title="Open-source movement">open-source movement</a> has inspired increased <a class="mw-redirect" href="/wiki/Transparency_(humanities)" title="Transparency (humanities)">transparency</a> and liberty in <a href="/wiki/Biotechnology" title="Biotechnology">biotechnology</a> research, for example by <a href="/wiki/CAMBIA" title="CAMBIA">CAMBIA</a>. Even the research methodologies themselves can benefit from the application of open-source principles. It has also given rise to the rapidly expanding <a href="/wiki/Open-source_hardware" title="Open-source hardware">open-source hardware</a> movement. In the book <i><a class="new" href="/w/index.php?title=Democratizing_Innovation&amp;action=edit&amp;redlink=1" title="Democratizing Innovation (page does not exist)">Democratizing Innovation</a></i> it is argued that a trend toward democratized innovation in physical products (e.g. open-source hardware) is occurring like the free and open-source software movement, and that the difference between <a href="/wiki/Crowdsourcing" title="Crowdsourcing">crowdsourcing</a> and open source is that open-source production is a cooperative activity initiated and voluntarily undertaken by members of the public. One of the primary geographically diverse communities that is utilizing this developmental method is the scientific community, for example using open-source hardware to reduce the cost of scientific equipment.</p>
<h3><span class="mw-headline" id="Computer_software">Computer software</span></h3>
<div class="hatnote" role="note">Main article: <a class="mw-redirect" href="/wiki/Free_and_open_source_software" title="Free and open source software">Free and open source software</a></div>


<p><a href="/wiki/Open-source_software" title="Open-source software">Open-source software</a> is software whose source code is published and made available to the public, enabling anyone to copy, modify and redistribute the source code without paying royalties or fees. Open-source code can evolve through community cooperation. These communities are composed of individual programmers as well as large companies. Some of the individual programmers who start an open-source project may end up establishing companies offering products or services incorporating open-source programs. Examples of open-source software products are:</p>
<h3><span class="mw-headline" id="Electronics">Electronics</span></h3>

<p><a href="/wiki/Open-source_hardware" title="Open-source hardware">Open-source hardware</a> is hardware whose initial specification, usually in a software format, are published and made available to the public, enabling anyone to copy, modify and redistribute the hardware and source code without paying royalties or fees. Open-source hardware evolves through community cooperation. These communities are composed of individual hardware/software developers, hobbyists, as well as very large companies. Examples of open-source hardware initiatives are:</p>
<ul>
<li><a href="/wiki/Openmoko" title="Openmoko">Openmoko</a>: a family of open-source <a href="/wiki/Mobile_phone" title="Mobile phone">mobile phones</a>, including the hardware specification and the <a href="/wiki/Operating_system" title="Operating system">operating system</a>.</li>
<li><a href="/wiki/OpenRISC" title="OpenRISC">OpenRISC</a>: an open-source microprocessor family, with architecture specification licensed under <a class="mw-redirect" href="/wiki/GNU_GPL" title="GNU GPL">GNU GPL</a> and implementation under <a class="mw-redirect" href="/wiki/LGPL" title="LGPL">LGPL</a>.</li>
<li><a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun Microsystems</a>'s <a href="/wiki/OpenSPARC" title="OpenSPARC">OpenSPARC</a> T1 Multicore processor. Sun has released it under GPL.</li>
<li><a href="/wiki/Arduino" title="Arduino">Arduino</a>, a microcontroller platform for hobbyists, artists and designers.</li>
<li><a class="new" href="/w/index.php?title=GizmoSphere&amp;action=edit&amp;redlink=1" title="GizmoSphere (page does not exist)">GizmoSphere</a>, an open-source development platform for the embedded design community; the site includes code downloads and hardware schematics along with free user guides, spec sheets and other documentation.</li>
<li><a href="/wiki/Simputer" title="Simputer">Simputer</a>, an open hardware <a class="mw-redirect" href="/wiki/Handheld_computer" title="Handheld computer">handheld computer</a>, designed in <a href="/wiki/India" title="India">India</a> for use in environments where computing devices such as personal computers are deemed inappropriate.</li>
<li><a href="/wiki/LEON" title="LEON">LEON</a>: A family of open-source microprocessors distributed in a library with peripheral <a class="mw-redirect" href="/wiki/IP_core" title="IP core">IP cores</a>, open <a href="/wiki/SPARC" title="SPARC">SPARC</a> V8 specification, implementation available under <a class="mw-redirect" href="/wiki/GNU_GPL" title="GNU GPL">GNU GPL</a>.</li>
<li><a href="/wiki/Tinkerforge" title="Tinkerforge">Tinkerforge</a>: A system of open-source stackable microcontroller building blocks. Allows to control motors and read out sensors with the programming languages C, C++, C#, Object Pascal, Java, PHP, Python and Ruby over a USB or Wifi connection on Windows, Linux and Mac OS X. All of the hardware is licensed under <a href="/wiki/CERN_Open_Hardware_License" title="CERN Open Hardware License">CERN OHL</a> (CERN Open Hardware License).</li>
<li><a href="/wiki/Open_Compute_Project" title="Open Compute Project">Open Compute Project</a>: designs for computer data center including power supply, Intel motherboard, AMD motherboard, chassis, racks, battery cabinet, and aspects of electrical and mechanical design.</li>
<li><a href="/wiki/Lasersaur" title="Lasersaur">Lasersaur</a>, an open-source <a class="mw-redirect" href="/wiki/Laser_cutter" title="Laser cutter">laser cutter</a>.</li>
</ul>
<h3><span class="mw-headline" id="Food_and_beverages">Food and beverages</span></h3>


<p>Some publishers of <a class="mw-redirect" href="/wiki/Open-access" title="Open-access">open-access</a> journals have argued that <a href="/wiki/Open_data" title="Open data">data</a> from <a href="/wiki/Food_science" title="Food science">food science</a> and <a href="/wiki/Gastronomy" title="Gastronomy">gastronomy</a> studies should be freely available to aid <a href="/wiki/Reproducibility" title="Reproducibility">reproducibility</a>. A number of people have published creative commons licensed recipe books.</p>
<ul>
<li><a href="/wiki/Open-source_cola" title="Open-source cola">Open-source colas</a> – cola soft drinks, similar to <a href="/wiki/Coca-Cola" title="Coca-Cola">Coca-Cola</a> and <a href="/wiki/Pepsi" title="Pepsi">Pepsi</a>, whose recipe is open source and developed by volunteers. The taste is said to be comparable to that of the standard beverages. Most corporations producing beverages hold their formulas as closely guarded secrets.</li>
<li><a href="/wiki/Free_Beer" title="Free Beer">Free Beer</a> (originally <i>Vores Øl</i>) – is an open-source beer created by students at the <a href="/wiki/IT_University_of_Copenhagen" title="IT University of Copenhagen">IT-University</a> in <a href="/wiki/Copenhagen" title="Copenhagen">Copenhagen</a> together with <a href="/wiki/Superflex" title="Superflex">Superflex</a>, an artist collective, to illustrate how open-source concepts might be applied outside the digital world.</li>
<li>In 2002, the beer company <a class="new" href="/w/index.php?title=Brewtopia&amp;action=edit&amp;redlink=1" title="Brewtopia (page does not exist)">Brewtopia</a> in Australia started an open-source brewery and invited the general population to be involved in the development and ownership of the brewery, and to vote on the development of every aspect of its beer, Blowfly, and its road to market. In return for their feedback and input, individuals received shares in the company, which is now publicly traded on a stock exchange in Australia. The company has always adhered to its open-source roots and is the only beer company in the world that allows the public to design, customise and develop its own beers online.</li>
</ul>
<h3><span class="mw-headline" id="Digital_content">Digital content</span></h3>

<ul>
<li><a class="external text" href="http://wikimediafoundation.org/wiki/Our_projects">Open-content projects organized by the Wikimedia Foundation</a> – Sites such as Wikipedia and Wiktionary have embraced the open-content <a href="/wiki/Creative_Commons" title="Creative Commons">Creative Commons</a> content licenses. These licenses were designed to adhere to principles similar to various open-source software development licenses. Many of these licenses ensure that content remains free for re-use, that source documents are made readily available to interested parties, and that changes to content are accepted easily back into the system. Important sites embracing open-source-like ideals are <a href="/wiki/Project_Gutenberg" title="Project Gutenberg">Project Gutenberg</a> and <a href="/wiki/Wikisource" title="Wikisource">Wikisource</a>, both of which post many books on which the copyright has expired and are thus in the <a href="/wiki/Public_domain" title="Public domain">public domain</a>, ensuring that anyone has free, unlimited access to that content.</li>
</ul>

<ul>
<li><a class="mw-redirect" href="/wiki/Open_ICEcat" title="Open ICEcat">Open ICEcat</a> is an open catalog for the IT, CE and Lighting sectors with product data-sheets based on <a href="/wiki/Open_Content_License" title="Open Content License">Open Content License</a> agreement. The digital content are distributed in XML and URL formats.</li>

<h2><span class="mw-headline" id="Society_and_culture">Society and culture</span></h2>
<p>Open-source culture is the creative practice of appropriation and free sharing of found and created content. Examples include <a href="/wiki/Collage" title="Collage">collage</a>, <a href="/wiki/Found_footage_(appropriation)" title="Found footage (appropriation)">found footage</a> film, music, and <a class="mw-redirect" href="/wiki/Appropriation_art" title="Appropriation art">appropriation art</a>. Open-source culture is one in which fixations, works entitled to copyright protection, are made generally available. Participants in the culture can modify those products and redistribute them back into the community or other organizations.</p>
<p>The rise of open-source culture in the 20th century resulted from a growing tension between creative practices that involve appropriation, and therefore require access to content that is often <a href="/wiki/Copyright" title="Copyright">copyrighted</a>, and increasingly restrictive intellectual property laws and policies governing access to copyrighted content. The two main ways in which intellectual property laws became more restrictive in the 20th century were extensions to the term of copyright (particularly in the <a href="/wiki/United_States" title="United States">United States</a>) and penalties, such as those articulated in the <a href="/wiki/Digital_Millennium_Copyright_Act" title="Digital Millennium Copyright Act">Digital Millennium Copyright Act</a> (DMCA), placed on attempts to circumvent anti-piracy technologies.</p>
<p>Although artistic appropriation is often permitted under <a href="/wiki/Fair_use" title="Fair use">fair-use</a> doctrines, the complexity and ambiguity of these doctrines creates an atmosphere of uncertainty among cultural practitioners. Also, the protective actions of copyright owners create what some call a "<a class="mw-redirect" href="/wiki/Chilling_effect_(term)" title="Chilling effect (term)">chilling effect</a>" among cultural practitioners.</p>
<p>In the late 20th century, cultural practitioners began to adopt the intellectual property licensing techniques of free software and open-source software to make their work more freely available to others, including the <a href="/wiki/Creative_Commons" title="Creative Commons">Creative Commons</a>.</p>
<p>The idea of an "open-source" culture runs parallel to "<a class="mw-redirect" href="/wiki/Free_Culture_movement" title="Free Culture movement">Free Culture</a>," but is substantively different. <i>Free culture</i> is a term derived from the <a href="/wiki/Free_software_movement" title="Free software movement">free software movement</a>, and in contrast to that vision of culture, proponents of open-source culture (OSC) maintain that some intellectual property law needs to exist to protect cultural producers. Yet they propose a more nuanced position than corporations have traditionally sought. Instead of seeing intellectual property law as an expression of instrumental rules intended to uphold either natural rights or desirable outcomes, an argument for OSC takes into account diverse goods (as in "the Good life") and ends.</p>
<p>One way of achieving the goal of making the fixations of cultural work generally available is to maximally utilize technology and <a href="/wiki/Digital_media" title="Digital media">digital media</a>. In keeping with <a href="/wiki/Moore%27s_law" title="Moore's law">Moore's law</a>'s prediction about processors, the cost of digital media and storage plummeted in the late 20th century. Consequently, the <a href="/wiki/Marginal_cost" title="Marginal cost">marginal cost</a> of digitally duplicating anything capable of being transmitted via digital media dropped to near zero. Combined with an explosive growth in <a href="/wiki/Personal_computer" title="Personal computer">personal computer</a> and technology ownership, the result is an increase in general population's access to digital media. This phenomenon facilitated growth in open-source culture because it allowed for rapid and inexpensive duplication and distribution of culture. Where the access to the majority of culture produced prior to the advent of digital media was limited by other constraints of proprietary and potentially "open" mediums, digital media is the latest technology with the potential to increase access to cultural products. Artists and users who choose to distribute their work digitally face none of the physical limitations that traditional cultural producers have been typically faced with. Accordingly, the audience of an open-source culture faces little physical cost in acquiring digital media.</p>
<p>Open-source culture precedes <a href="/wiki/Richard_Stallman" title="Richard Stallman">Richard Stallman</a>'s codification of free software with the creation of the <a class="mw-redirect" href="/wiki/Free_Software_movement" title="Free Software movement">Free Software movement</a>. As the public began to communicate through <a class="mw-redirect" href="/wiki/Bulletin_Board_Systems" title="Bulletin Board Systems">Bulletin Board Systems</a> (BBS) like <a href="/wiki/FidoNet" title="FidoNet">FidoNet</a>, places like Sourcery Systems BBS were dedicated to providing source code to <a class="mw-redirect" href="/wiki/Public_Domain" title="Public Domain">Public Domain</a>, <a href="/wiki/Shareware" title="Shareware">Shareware</a> and <a href="/wiki/Freeware" title="Freeware">Freeware</a> programs.</p>
<p>Essentially born out of a desire for increased general access to digital media, <a href="/wiki/Internet" title="Internet">the Internet</a> is open-source culture's most valuable asset. It is questionable whether the goals of an open-source culture could be achieved without the Internet. The global network not only fosters an environment where culture can be generally accessible, but also allows for easy and inexpensive redistribution of culture back into various communities. Some reasons for this are as follows.</p>
<p>First, the Internet allows even greater access to inexpensive digital media and storage. Instead of users being limited to their own facilities and resources, they are granted access to a vast network of facilities and resources, some free. Sites such as <a href="/wiki/CcMixter" title="CcMixter">ccMixter</a> offer up free web space for anyone willing to license their work under a <a href="/wiki/Creative_Commons" title="Creative Commons">Creative Commons</a> license. The resulting cultural product is then available to download free (generally accessible) to anyone with an Internet connection. Second, users are granted unprecedented access to each other. Older analog technologies such as the <a href="/wiki/Telephone" title="Telephone">telephone</a> or <a href="/wiki/Television" title="Television">television</a> have limitations on the kind of interaction users can have. In the case of television there is little, if any interaction between users participating on the network. And in the case of the telephone, users rarely interact with any more than a couple of their known peers. On the Internet, however, users have the potential to access and meet millions of their peers. This aspect of the Internet facilitates the modification of culture as users are able to collaborate and communicate with each other across international and cultural boundaries. The speed in which digital media travels on the Internet in turn facilitates the redistribution of culture.</p>
<p>Through various technologies such as <a href="/wiki/Peer-to-peer" title="Peer-to-peer">peer-to-peer</a> networks and <a class="mw-redirect" href="/wiki/Blogs" title="Blogs">blogs</a>, cultural producers can take advantage of vast <a class="mw-redirect" href="/wiki/Social_networks" title="Social networks">social networks</a> to distribute their products. As opposed to traditional media distribution, redistributing digital media on the Internet can be virtually costless. Technologies such as <a class="mw-redirect" href="/wiki/BitTorrent_(protocol)" title="BitTorrent (protocol)">BitTorrent</a> and <a href="/wiki/Gnutella" title="Gnutella">Gnutella</a> take advantage of various characteristics of the Internet protocol (<a class="mw-redirect" href="/wiki/TCP/IP" title="TCP/IP">TCP/IP</a>) in an attempt to totally decentralize file distribution.</p>

<h3><span class="mw-headline" id="Media">Media</span></h3>
<p><a href="/wiki/Open-source_journalism" title="Open-source journalism">Open-source journalism</a> formerly referred to the standard journalistic techniques of news gathering and fact checking, reflecting <i><a href="/wiki/Open-source_intelligence" title="Open-source intelligence">open-source intelligence</a></i> a similar term used in military intelligence circles. Now, <i>open-source journalism</i> commonly refers to forms of innovative publishing of <a class="mw-redirect" href="/wiki/Online_journalism" title="Online journalism">online journalism</a>, rather than the sourcing of news stories by a professional journalist. In the December 25, 2006 issue of TIME magazine this is referred to as <a class="mw-redirect" href="/wiki/User_generated_content" title="User generated content">user created content</a> and listed alongside more traditional open-source projects such as <a href="/wiki/OpenSolaris" title="OpenSolaris">OpenSolaris</a> and <a href="/wiki/Linux" title="Linux">Linux</a>.</p>
<p><a href="/wiki/Blog" title="Blog">Weblogs</a>, or blogs, are another significant platform for open-source culture. Blogs consist of periodic, reverse chronologically ordered posts, using a technology that makes webpages easily updatable with no understanding of design, code, or <a href="/wiki/File_transfer" title="File transfer">file transfer</a> required. While corporations, political campaigns and other formal institutions have begun using these tools to distribute information, many blogs are used by individuals for personal expression, political organizing, and socializing. Some, such as <a href="/wiki/LiveJournal" title="LiveJournal">LiveJournal</a> or <a href="/wiki/WordPress" title="WordPress">WordPress</a>, utilize open-source software that is open to the public and can be modified by users to fit their own tastes. Whether the code is open or not, this format represents a nimble tool for people to borrow and re-present culture; whereas traditional websites made the illegal reproduction of culture difficult to regulate, the mutability of blogs makes "open sourcing" even more uncontrollable since it allows a larger portion of the population to replicate material more quickly in the public sphere.</p>
<p><a class="mw-redirect" href="/wiki/Messageboards" title="Messageboards">Messageboards</a> are another platform for open-source culture. Messageboards (also known as discussion boards or forums), are places online where people with similar interests can congregate and post messages for the community to read and respond to. Messageboards sometimes have moderators who enforce community standards of etiquette such as banning users who are <a href="/wiki/Spamming" title="Spamming">spammers</a>. Other common board features are private messages (where users can send messages to one another) as well as chat (a way to have a real time conversation online) and image uploading. Some messageboards use <a href="/wiki/PhpBB" title="PhpBB">phpBB</a>, which is a free open-source package. Where blogs are more about individual expression and tend to revolve around their authors, messageboards are about creating a conversation amongst its users where information can be shared freely and quickly. Messageboards are a way to remove intermediaries from everyday life—for instance, instead of relying on commercials and other forms of advertising, one can ask other users for frank reviews of a product, movie or CD. By removing the cultural middlemen, messageboards help speed the flow of information and exchange of ideas.</p>
<p><a href="/wiki/OpenDocument" title="OpenDocument">OpenDocument</a> is an <a href="/wiki/Open_format" title="Open format">open</a> <a href="/wiki/Document_file_format" title="Document file format">document file format</a> for saving and exchanging editable office documents such as text documents (including memos, reports, and books), <a href="/wiki/Spreadsheet" title="Spreadsheet">spreadsheets</a>, charts, and presentations. Organizations and individuals that store their data in an open format such as OpenDocument avoid being <a href="/wiki/Vendor_lock-in" title="Vendor lock-in">locked into</a> a single software vendor, leaving them free to switch software if their current vendor goes out of business, raises their prices, changes their software, or changes their <a href="/wiki/Software_license" title="Software license">licensing</a> terms to something less favorable.</p>
<p><a href="/wiki/Open-source_film" title="Open-source film">Open-source movie production</a> is either an open call system in which a changing crew and cast collaborate in movie production, a system in which the end result is made available for re-use by others or in which exclusively open-source products are used in the production. The 2006 movie <a href="/wiki/Elephants_Dream" title="Elephants Dream">Elephants Dream</a> is said to be the "world's first open movie", created entirely using <a class="mw-redirect" href="/wiki/Open-source_technology" title="Open-source technology">open-source technology</a>.</p>
<p>An open-source documentary film has a production process allowing the open contributions of archival material <a href="/wiki/Footage" title="Footage">footage</a>, and other filmic elements, both in unedited and edited form, similar to crowdsourcing. By doing so, on-line contributors become part of the process of creating the film, helping to influence the editorial and visual material to be used in the documentary, as well as its thematic development. The first open-source documentary film is the non-profit "<a class="mw-redirect" href="/wiki/The_American_Revolution" title="The American Revolution">The American Revolution</a>", which went into development in 2006, and will examine the role media played in the cultural, social and political changes from 1968 to 1974 through the story of radio station WBCN-FM in Boston. The film is being produced by Lichtenstein Creative Media and the non-profit Filmmakers Collaborative. Open Source Cinema is a website to create Basement Tapes, a feature documentary about copyright in the digital age, co-produced by the National Film Board of Canada. <a href="/wiki/Open-source_film" title="Open-source film">Open-source film-making</a> refers to a form of film-making that takes a method of idea formation from open-source software, but in this case the 'source' for a filmmaker is raw unedited footage rather than programming code. It can also refer to a method of film-making where the process of creation is 'open' i.e. a disparate group of contributors, at different times contribute to the final piece.</p>
<p><a class="new" href="/w/index.php?title=Open-IPTV&amp;action=edit&amp;redlink=1" title="Open-IPTV (page does not exist)">Open-IPTV</a> is <a href="/wiki/IPTV" title="IPTV">IPTV</a> that is not limited to one recording studio, production studio, or cast. Open-IPTV uses the Internet or other means to pool efforts and resources together to create an online community that all contributes to a show.</p>
<h3><span class="mw-headline" id="Education">Education</span></h3>

<p>Within the academic community, there is discussion about expanding what could be called the "intellectual commons" (analogous to the <a href="/wiki/Creative_Commons" title="Creative Commons">Creative Commons</a>). Proponents of this view have hailed the <a class="mw-redirect" href="/wiki/Connexions" title="Connexions">Connexions</a> Project at <a href="/wiki/Rice_University" title="Rice University">Rice University</a>, <a href="/wiki/MIT_OpenCourseWare" title="MIT OpenCourseWare">OpenCourseWare</a> project at <a href="/wiki/Massachusetts_Institute_of_Technology" title="Massachusetts Institute of Technology">MIT</a>, <a href="/wiki/Eugene_Thacker" title="Eugene Thacker">Eugene Thacker</a>'s article on "open-source DNA", the "Open Source Cultural Database", <a class="mw-redirect" href="/wiki/Salman_Khan_(educator)" title="Salman Khan (educator)">Salman Khan</a>'s <a href="/wiki/Khan_Academy" title="Khan Academy">Khan Academy</a> and <a href="/wiki/Wikipedia" title="Wikipedia">Wikipedia</a> as examples of applying open source outside the realm of computer software.</p>
<p><a href="/wiki/Open-source_curriculum" title="Open-source curriculum">Open-source curricula</a> are instructional resources whose digital source can be freely used, distributed and modified.</p>
<p>Another strand to the academic community is in the area of research. Many funded research projects produce software as part of their work. There is an increasing interest in making the outputs of such projects available under an open-source license. In the UK the <a href="/wiki/Jisc" title="Jisc">Joint Information Systems Committee (JISC)</a> has developed a policy on open-source software. JISC also funds a development service called <a href="/wiki/OSS_Watch" title="OSS Watch">OSS Watch</a> which acts as an advisory service for higher and further education institutions wishing to use, contribute to and develop open-source software.</p>
<p>On March 30, 2010, President Barack Obama signed the Health Care and Education Reconciliation Act, which included $2 billion over four years to fund the TAACCCT program, which is described as "the largest OER (open education resources) initiative in the world and uniquely focused on creating curricula in partnership with industry for credentials in vocational industry sectors like manufacturing, health, energy, transportation, and IT".</p>
<h3><span class="mw-headline" id="Innovation_communities">Innovation communities</span></h3>
<p>The principle of sharing pre-dates the open-source movement; for example, the free sharing of information has been institutionalized in the scientific enterprise since at least the 19th century. Open-source principles have always been part of the scientific community. The sociologist <a href="/wiki/Robert_K._Merton" title="Robert K. Merton">Robert K. Merton</a> described the four basic elements of the community—universalism (an international perspective), communalism (sharing information), disinterestedness (removing one's personal views from the scientific inquiry) and organized skepticism (requirements of proof and review) that accurately describe the scientific community today.</p>
<p>These principles are, in part, complemented by US law's focus on protecting expression and method but not the ideas themselves. There is also a tradition of publishing research results to the scientific community instead of keeping all such knowledge proprietary. One of the recent initiatives in scientific publishing has been <a class="mw-redirect" href="/wiki/Open_access_(publishing)" title="Open access (publishing)">open access</a>—the idea that research should be published in such a way that it is free and available to the public. There are currently many open access journals where the information is available free online, however most journals do charge a fee (either to users or libraries for access). The <a href="/wiki/Budapest_Open_Access_Initiative" title="Budapest Open Access Initiative">Budapest Open Access Initiative</a> is an international effort with the goal of making all research articles available free on the Internet.</p>
<p>The <a href="/wiki/National_Institutes_of_Health" title="National Institutes of Health">National Institutes of Health</a> has recently proposed a policy on "Enhanced Public Access to NIH Research Information". This policy would provide a free, searchable resource of NIH-funded results to the public and with other international repositories six months after its initial publication. The NIH's move is an important one because there is significant amount of public funding in scientific research. Many of the questions have yet to be answered—the balancing of profit vs. public access, and ensuring that desirable standards and incentives do not diminish with a shift to open access.</p>
<p>Farmavita.Net is a community of pharmaceuticals executives that has recently proposed a new business model of open-source pharmaceuticals. The project is targeted to development and sharing of know-how for manufacture of essential and life-saving medicines. It is mainly dedicated to the countries with less developed economies where local pharmaceutical research and development resources are insufficient for national needs. It will be limited to generic (off-patent) medicines with established use. By the definition, medicinal product have a "well-established use" if is used for at least 15 years, with recognized efficacy and an acceptable level of safety. In that event, the expensive clinical test and trial results could be replaced by appropriate scientific literature.</p>
<p><a href="/wiki/Benjamin_Franklin" title="Benjamin Franklin">Benjamin Franklin</a> was an early contributor eventually donating all his inventions including the <a href="/wiki/Franklin_stove" title="Franklin stove">Franklin stove</a>, <a href="/wiki/Bifocals" title="Bifocals">bifocals</a>, and the <a href="/wiki/Lightning_rod" title="Lightning rod">lightning rod</a> to the public domain.</p>
<p>New NGO communities are starting to use the open-source technology as a tool. One example is the Open Source Youth Network started in 2007 in Lisboa by ISCA members.</p>
<p><a href="/wiki/Open_innovation" title="Open innovation">Open innovation</a> is also a new emerging concept which advocate putting R&amp;D in a common pool. The <a href="/wiki/Eclipse_(software)" title="Eclipse (software)">Eclipse</a> platform is openly presenting itself as an Open innovation network.</p>
<h3><span class="mw-headline" id="Arts_and_recreation">Arts and recreation</span></h3>
<p>Copyright protection is used in the performing arts and even in athletic activities. Some groups have attempted to remove copyright from such practices.</p>
<p>In 2012, Russian music composer, scientist and Russian Pirate Party member Victor Argonov presented detailed raw files of his electronic opera "2032" under free license <a href="/wiki/Creative_Commons_license" title="Creative Commons license">CC-BY-NC 3.0</a>. This opera was originally composed and published in 2007 by Russian label <a href="/wiki/MC_Entertainment" title="MC Entertainment">MC Entertainment</a> as a commercial product, but then the author changed its status to free. In his blog <a class="external autonumber" href="http://argonov.livejournal.com" rel="nofollow">[2]</a> he said that he decided to open raw files (including wav, midi and other used formats) to the public in order to support worldwide pirate actions against <a class="mw-redirect" href="/wiki/SOPA" title="SOPA">SOPA</a> and <a class="mw-redirect" href="/wiki/Personal_Information_Protection_Act" title="Personal Information Protection Act">PIPA</a>. Several Internet resources, called "2032" the first open-source musical opera in history.</p>
<h3><span class="mw-headline" id="Other_related_movements">Other related movements</span></h3>

<p>The following are events and applications that have been developed via the open source community as and echo the ideologies of the open source movement.</p>
<p><b><a href="/wiki/OpenCourseWare" title="OpenCourseWare">Open Education Consortium</a></b> — an organization composed of various colleges that support open source and share some of their material online. This organization, headed by <a href="/wiki/Massachusetts_Institute_of_Technology" title="Massachusetts Institute of Technology">Massachusetts Institute of Technology</a>, was established to aid in the exchange of open source educational materials.</p>
<p><b><a href="/wiki/Wikipedia" title="Wikipedia">Wikipedia</a></b> — user-generated <a class="mw-redirect" href="/wiki/Internet_encyclopedia_project" title="Internet encyclopedia project">online encyclopedia</a> with sister projects in academic areas, such as <a href="/wiki/Wikiversity" title="Wikiversity">Wikiversity</a> — a community dedicated to the creation and exchange of learning materials</p>
<p><b><a href="/wiki/Project_Gutenberg" title="Project Gutenberg">Project Gutenberg</a></b> — prior to the existence of <a href="/wiki/Google_Scholar" title="Google Scholar">Google Scholar</a> Beta, this was the first supplier of <a href="/wiki/E-book" title="E-book">electronic books</a> and the very first free library project</p>
<p><b>Synthetic Biology</b>- Synthetic Biology is considered the feasibility of the open source movement. This new technology is important and exciting because it promises to enable cheap, lifesaving new drugs as well as helping to yield biofuels that may help to solve our energy problem. Although synthetic biology has not yet come out of its "lab" stage, it has great potential to become industrialized in the near future. In order to industrialize open source science, there are some scientists who are trying to build their own brand of it.</p>
<h3><span class="mw-headline" id="Ideologically_related_movements">Ideologically related movements</span></h3>
<p>The <a class="mw-redirect" href="/wiki/Open_access_movement" title="Open access movement">open access movement</a> is a movement that is similar in ideology to the open source movement. Members of this movement maintain that academic material should be readily available to provide help with “future research, assist in teaching and aid in academic purposes.” The Open access movement aims to eliminate subscription fees and licensing restrictions of academic materials</p>
<p>The <a href="/wiki/Free_culture_movement" title="Free culture movement">free culture movement</a> is a movement that seeks to achieve a culture that engages in collective freedom via freedom of expression, free public access to knowledge and information, full demonstration of creativity and innovation in various arenas and promotion of citizen liberties.</p>
<p><a href="/wiki/Creative_Commons" title="Creative Commons">Creative Commons</a> is an organization that “develops, supports, and stewards legal and technical infrastructure that maximizes digital creativity, sharing, and innovation.” It encourages the use of protected properties online for research, education, and creative purposes in pursuit of a universal access. Creative Commons provides an infrastructure through a set of copyright licenses and tools that creates a better balance within the realm of “all rights reserved” properties. The Creative Commons license offers a slightly more lenient alternative to “all rights reserved” copyrights for those who do not wish to exclude the use of their material.</p>
<p><a href="/wiki/The_Zeitgeist_Movement" title="The Zeitgeist Movement">The Zeitgeist Movement</a> is an international social movement that advocates a transition into a <a href="/wiki/Sustainability" title="Sustainability">sustainable</a> "resource-based economy" based on <a href="/wiki/Collaboration" title="Collaboration">collaboration</a> in which monetary incentives are replaced by commons-based ones with everyone <a href="/wiki/Sharing_economy" title="Sharing economy">having access</a> to everything (from code to products) as in "open source everything". While its activism and events are typically focused on media and education, TZM is a major supporter of open source projects worldwide since they allow for uninhibited advancement of science and technology, independent of constraints posed by institutions of patenting and capitalist investment.</p>
<p><a href="/wiki/P2P_Foundation" title="P2P Foundation">P2P Foundation</a> is an “international organization focused on studying, researching, documenting and promoting <a href="/wiki/Social_peer-to-peer_processes" title="Social peer-to-peer processes">peer to peer practices</a> in a very broad sense”. Its objectives incorporate those of the open source movement, whose principles are integrated in a larger socio-economic model.</p>
